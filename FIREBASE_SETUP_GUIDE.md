# Firebase 超入門ガイド（このゲームを動かすために必要な作業だけ）

「Firebaseの使い方が分からない」という方向けに、**この画面をこう押す**というレベルまで
分解して説明します。上から順番にやれば動くようになります。難しい言葉の説明は最後に
まとめてあるので、まずは手順だけ追ってもらって大丈夫です。

全体像だけ先に：
> Firebase = Googleが提供する「ログイン機能」と「データベース」をまとめて borrow できるサービス。
> あなたはこのゲームのためだけの専用の“棚”（プロジェクト）を1つ作り、そこに住所（設定値）を
> ゲームのコードに教えてあげる、というのが今回やることの全てです。

---

## 手順0：準備するもの

- Googleアカウント（Gmailなど）だけでOKです。無料で使えます（クレジットカード登録も不要です）。

---

## 手順1：Firebaseプロジェクトを作る

1. ブラウザで https://console.firebase.google.com/ を開く
2. Googleアカウントでログイン
3. 「プロジェクトを作成」（or「+ プロジェクトを追加」）をクリック
4. プロジェクト名を入力（例：`slot-sugoroku`）→「続行」
5. Googleアナリティクスの設定画面が出ますが、今回は不要なので
   「このプロジェクトでGoogleアナリティクスを有効にする」のチェックを**外して**「プロジェクトを作成」
6. 数十秒待つと「準備ができました」と出るので「続行」

これで「棚」が1つできました。

---

## 手順2：ウェブアプリを登録して「設定値」をもらう

ここで、ゲーム側の `js/firebase-config.js` に貼り付ける値をもらいます。

1. プロジェクトのトップ画面（Project Overviewの画面）にある **`</>`（ウェブ）のアイコン** をクリック
2. アプリのニックネームを入力（例：`slot-sugoroku-web`）
3. 「Firebase Hosting も設定します」のチェックは**外したままでOK**（後で必要になれば別途できます）
4. 「アプリを登録」をクリック
5. 画面に以下のような**コードの塊**が表示されます。これが今回欲しいものです。

```js
const firebaseConfig = {
  apiKey: "AIzaSy..........",
  authDomain: "slot-sugoroku-xxxxx.firebaseapp.com",
  projectId: "slot-sugoroku-xxxxx",
  storageBucket: "slot-sugoroku-xxxxx.appspot.com",
  messagingSenderId: "1234567890",
  appId: "1:1234567890:web:abcdef123456"
};
```

6. この `firebaseConfig` の中身をまるごとコピーして、プロジェクトの
   `js/firebase-config.js` を開き、`export const firebaseConfig = { ... }` の
   `{ ... }` の中身をこの値で置き換えて保存してください
   （`YOUR_API_KEY` などのプレースホルダー文字列を、実際の値に差し替えるイメージです）
7. 「コンソールに進む」をクリックして画面を閉じてOK

---

## 手順3：ログイン機能（Authentication）を有効にする

このゲームでは、プレイヤーは「匿名ログイン」、管理者は「メール/パスワードログイン」を使います。

1. 左メニューの「構築」→「Authentication」をクリック
2. 「始める」をクリック
3. 「Sign-in method」タブを開く
4. 一覧から **「匿名」** をクリック → 右上のスイッチを **有効** にする → 「保存」
5. 同じ一覧から **「メール／パスワード」** をクリック → 一番上のスイッチを **有効** にする → 「保存」
   （下の「メールリンク」は無効のままでOK）

---

## 手順4：データベース（Firestore）を作る

1. 左メニューの「構築」→「Firestore Database」をクリック
2. 「データベースの作成」をクリック
3. ロケーション（サーバーの地域）を選ぶ画面が出ます。日本からのアクセスが多いなら
   `asia-northeast1`（東京）などを選ぶと良いです → 「次へ」
4. セキュリティルールの初期設定を聞かれたら、どちらを選んでもOKです
   （後の手順5でこちらが用意したルールに丸ごと上書きするため）→「有効にする」
5. 数十秒待つとデータベースの画面が開きます

---

## 手順5：セキュリティルールを貼り付ける

これは「誰が・何を読み書きしていいか」のルールです。プロジェクトに同梱している
`firestore.rules` の中身をそのまま使います。

1. Firestore Databaseの画面上部にある「ルール」タブをクリック
2. 表示されているエディタの中身を全て選択して削除
3. プロジェクト内の `firestore.rules` ファイルの中身を全部コピーして貼り付け
4. 右上の「公開」をクリック

---

## 手順6：管理者アカウントを作る

管理画面（`admin.html`）にログインするための、あなた専用のアカウントです。

1. 「Authentication」→「Users」タブ→「ユーザーを追加」
2. メールアドレスとパスワードを決めて入力（実在するメールでなくても、
   形式が正しければ登録できます。例：`admin@example.com` / 好きなパスワード8文字以上）
3. 「ユーザーを追加」をクリックすると、一覧に1行追加されます
4. その行に表示されている「ユーザーUID」という文字列（英数字の羅列）をコピー
   （長いので、メモ帳などに一旦貼り付けておくと安心です）

次に、そのUIDを「管理者です」という印としてFirestoreに登録します。

5. 「Firestore Database」→「データ」タブに戻る
6. 「コレクションを開始」をクリック
7. コレクションID欄に `admins` と入力 →「次へ」
8. ドキュメントIDの欄に、先ほどコピーした**UIDをそのまま貼り付け**
9. フィールドは何も入力しなくてOK（フィールド名・値ともに空欄のままだとエラーになる場合は、
   フィールド名に `note`、値（文字列）に `admin` とでも入れておけばOK）
10. 「保存」をクリック

これで `admins/そのUID` というドキュメントができ、そのアカウントだけが管理画面に
入れるようになります。

---

## 手順7：ローカルで動かしてみる

このゲームは `import`/`export` という書き方（ESモジュール）を使っているため、
`index.html` を**ダブルクリックで直接開いても動きません**（ブラウザの安全機能で止まります）。
簡易サーバーを使って開く必要があります。

### パソコンにNode.jsが入っている場合
1. ターミナル（コマンドプロンプト）でプロジェクトのフォルダに移動
2. 以下を実行
   ```
   npx serve .
   ```
3. 表示されたURL（例：`http://localhost:3000`）をブラウザで開く

### パソコンにPythonが入っている場合
1. ターミナルでプロジェクトのフォルダに移動
2. 以下を実行
   ```
   python3 -m http.server 8080
   ```
3. ブラウザで `http://localhost:8080` を開く

### どちらも無い場合
- VS Codeを使っているなら、拡張機能「Live Server」をインストールして
  `index.html` を右クリック→「Open with Live Server」が一番簡単です

これでゲーム画面が開けば、匿名ログイン→プロフィール自動作成が走り、
プレイできる状態になっているはずです。

---

## 手順8：管理画面を開いてみる

同じサーバーを使ったまま、ブラウザで `admin.html` を開いてください
（例：`http://localhost:8080/admin.html`）。手順6で作ったメールアドレスと
パスワードでログインできれば成功です。

---

## 手順9（任意）：インターネット上に公開する（Firebase Hosting）

自分のパソコンだけでなく、誰でもアクセスできるURLにしたい場合はFirebase Hostingが
一番簡単です。ターミナル操作が必要になります。

```bash
npm install -g firebase-tools   # Firebaseのコマンドラインツールをインストール
firebase login                  # ブラウザが開くのでGoogleアカウントでログイン
cd プロジェクトのフォルダ
firebase init hosting           # 質問にはだいたいEnterでOK。公開フォルダは「.」（今のフォルダ）でOK
firebase deploy                 # これでインターネットに公開される
```

デプロイが終わると `https://あなたのプロジェクトID.web.app` のようなURLが発行されます。

## よくあるつまずきポイント

| 症状 | 原因・対処 |
|---|---|
| 画面が真っ白、コンソールに`Failed to resolve module`等 | `index.html`を直接ダブルクリックしている可能性大。手順7のようにサーバー経由で開いてください |
| `Missing or insufficient permissions` というエラー | 手順5のセキュリティルールがまだ貼られていない／公開されていない可能性があります |
| 管理画面にログインできない／「権限がありません」と出る | 手順6の `admins/{UID}` の作成を忘れているか、UIDの貼り間違いの可能性があります |
| ランキングが表示されない | Firestoreの「データ」タブで `players` コレクションにデータができているか確認してください。1回もゲームを開いていないと何も無いのが正常です |
| コインやスコアの動きがおかしい | ブラウザの開発者ツール（F12キー）の「Console」タブにエラーが出ていないか確認してください |

## 言葉の意味（参考程度に）

- **Firebase**：Googleが提供する「ログイン」「データベース」などをまとめて借りられるサービス
- **プロジェクト**：Firebase上での「棚」1つぶんの単位。今回は1つだけ作ればOK
- **Authentication（認証）**：「誰がログインしているか」を管理する機能
- **匿名ログイン**：メールアドレス登録なしで、その端末・ブラウザ専用のIDが自動発行されるログイン方式。今回はプレイヤー識別に使っています
- **Firestore**：Googleが提供するデータベース。プロフィールやランキングなどのデータを保存する場所
- **セキュリティルール**：そのデータベースに対して「誰が・何を読み書きしていいか」を決める設定
- **UID**：ログインしているユーザー1人ひとりに割り振られる、世界に1つだけの識別番号
